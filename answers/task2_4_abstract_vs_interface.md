# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор         | Обоснование                                                                          |

| Некоторые объекты (документ, изображение, настройки) можно сериализовать | интерфейс         | отношение "умеет" (сериализация), множественная реализация                           |

| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi |          интерфейс         | отношение "умеет" (подлкючение к wifi), множественная реализация                     |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | абстрактный класс | Есть общие поля (цвет) - состояние, отношение "является"                             |
